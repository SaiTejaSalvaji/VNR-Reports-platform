# 18 Glossary

A reference for terms and concepts used throughout the VNR Reports codebase and documentation.

## General Technical Terms

* **SPA (Single Page Application)**: A web application (like React) that loads a single HTML page and dynamically updates the page as the user interacts with it, rather than loading entire new pages from the server.
* **JWT (JSON Web Token)**: An open standard for securely transmitting information between parties as a JSON object. Used here for stateless authentication.
* **RBAC (Role-Based Access Control)**: A method of restricting network access based on the roles of individual users within an enterprise (e.g., Admin, HOD, Faculty).
* **JSONB**: A PostgreSQL data type that stores JSON data in a decomposed binary format. It is slower to insert than raw text JSON but significantly faster to process and query because it supports indexing.
* **Serverless**: A cloud computing execution model where the cloud provider dynamically manages the allocation and provisioning of servers (e.g., Vercel, Firebase Functions).
* **Cold Start**: The delay that occurs when a serverless function is triggered after a period of inactivity, requiring the provider to spin up a new container to handle the request.

## Project Specific Terms

* **Section**: A specific category of report data, like "Faculty Publications" or "Guest Lectures". 
* **Section Key**: The unique string identifier for a section (e.g., `journal_publications`). Used heavily in URLs and database lookups.
* **Metadata Schema**: The JSON definition stored in the database that dictates what columns/inputs a specific Section should render on the frontend.
* **HOD**: Head of Department. A user role that has higher privileges than Faculty, allowing them to view and manage data for their entire specific department.
* **Reports-Incharge**: A delegated role functioning similarly to an HOD for the purpose of report compilation.
* **Global Month Context**: The specific Month and Year selected in the top navigation bar. The entire application filters its data based on this context.
* **Snapshot**: A massive, consolidated Excel or Word document generated by an Admin that aggregates data from all departments for a given time period.
* **Neon**: The serverless PostgreSQL provider hosting the database.
* **Pooler URL**: A specific database connection string provided by Neon that routes traffic through `PgBouncer`, preventing serverless functions from overwhelming the database with too many concurrent connections.
